This document describes the known bit allocations and message IDs for the BitTorrent protocol. To request a bit allocation contact editor@bittorrent.org. Contact the same address if you are aware of any omissions.

1. Reserved Bit Allocations 

 reserved[0]
 0x80  Azureus Messaging Protocol

 reserved[5]
 0x10  General Extension Support (implemented by libtorrent, uTorrent,...)

 reserved[7]
 0x01  BitTorrent DHT
 0x04  suggest, haveall, havenone, reject request, and allow fast extensions

2. Reserved Message IDs 

Core Protocol:
 0x00   choke
 0x01   unchoke
 0x02   interested
 0x03   not interested
 0x04   have
 0x05   bitfield
 0x06   request
 0x07   piece
 0x08   cancel
 0x09   port

2.1 Fast Extensions 

 0x0D   suggest
 0x0E   have all
 0x0F   have none
 0x10   reject request
 0x11   allowed fast

Additional IDs used in deployed clients 

 0x14   General Extension Handshake (implemented in libtorrent, uTorrent,...)
